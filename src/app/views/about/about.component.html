About the Project

The Angular project is a simple interest loan calculator that allows users to simulate loans for various purposes. It features a responsive interface and is crafted in pure CSS.

The project employs various Angular development techniques to create an interactive and informative user experience.

Technical Description

The project is an Angular application that follows best development practices and utilizes various advanced techniques. Here are the main components and techniques used:

  1. Angular Modules and Components:

    • `AppModule`: The root module of the Angular application.
    • `AppRoutingModule`: Routing module that manages application routes.
    • `AppComponent`: The root component of the application.
    • VIEWS:
      • `HomeComponent`: Home page component located in the `views/home` folder.
      • `AboutComponent`: "About" page component located in the `views/about` folder.
      • `LoanCalculatorComponent`: Loan calculator component located in the `views/loanCalculator` folder.
    • COMPONENTS:
      • `NavbarComponent`: Navigation bar component located in the `components/navbar` folder.
      • `FooterComponent`: Footer component located in the `components/footer` folder.
      • `GraphBarComponent`: Component that displays bar charts located in the `components/graphBar` folder.
    • MODEL:
      • `Graph`: An interface that defines the data structure used to create bar charts.
    • SERVICES:
      • `LoanCalculatorService`: A service that handles loan simulation. It makes HTTP calls to a simulation service.
    • db.json:
      • A JSON file that can contain simulated data or application information, used by the `LoanCalculatorService`.

  2. Routing:

    • Routing is configured in the `AppRoutingModule`, which defines routes to different components.

  3. HTML and CSS of Components:

    • Each component has its own HTML and CSS files to define the structure and style.
    • Styles are primarily defined using CSS classes.

  4. Services:

    • `LoanCalculatorService`: A service that handles loan simulation. It makes HTTP calls to a simulation service.

  5. Data Model:

    • `Graph`: An interface that defines the data structure used to create bar charts.

  6. Forms and Validation:

    • Forms are defined in components using the `FormBuilder` module.
    • Input validation is performed using Angular's validation features.

  7. Navigation Routing:

    • Page navigation is done using `routerLink` directives in the navigation bar component's links.

  8. Styling:

    • Styling is primarily done with CSS, with style definitions for components and interface elements.

  9. Data Communication:

    • The `LoanCalculatorComponent` communicates with the `LoanCalculatorService` to perform loan simulation and update displayed data.

  10. Bar Charts:

    • The `GraphBarComponent` is responsible for displaying bar charts with simulation data.

  11. Responsiveness:

    • Style rules are defined to make the application responsive on smaller screens.

  12. HTTP Requests:

    • The application makes HTTP calls to a simulated service to obtain loan simulation results.
